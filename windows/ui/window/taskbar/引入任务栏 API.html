引入任务栏 API
Yochay Kiriaty & Sasha Goldshtein

摘要

在 Windows 7 任务栏 是最新的一系列的入门 Windows 1.0 进化任务栏设计。 它表示设计和体系结构与用户的体验有关的更改，并提供了在用户的工作效率的重大改进。 图 1 显示了 Windows 1.0 任务栏。 （请注意 1.0 任务栏与 Windows Vista 任务栏比 Windows 7 任务栏将多个 resemblance）。

图 1 Windows 1.0 任务栏
从多个用户会话期间最近几年的分析表明启动图面 （专用于启动程序的那些区域） 的一个富有不一定使用户的工作任何更容易。 就是而言用户界面设计，以防止用户混淆最好通常将只有一种完成某项任务的方法。
Windows 7 任务栏的设计目标是将控件返回用户并支持下列原则最小化混淆：
  • 频繁的程序和目标的单个启动图面。 这有助于使这些应用程序和数据使用所有时间右侧伸手。
  • 轻松地控制在应用程序和窗口之间切换以及排列窗口进行简单且易于管理。
  • 骞插噣自由噪音、 简单。
较大的任务栏按钮、 视觉清晰度，和 sleekness 设计的制约用户体验，一旦登录到 Windows 7。 但是，如何开发人员可以利用 Windows 7 任务栏所提供的新功能？ 如何基础机制协同工作以便获得最佳用户体验？ 在本文中，我们将探讨新功能和支持本机代码和托管代码中的示例与 Windows 7 任务栏上的基本功能。


任务栏按钮
任务栏按钮提供了网关访问新的 Windows 7 任务栏的功能。 在最大，需要注意是任务栏也包含不当前正在运行的应用程序的图标。 渚嬪的方式  用户可能会固定到任务栏结果是一个固定的图标在任务栏上的应用程序，但是重应用程序没有运行。 单击该图标会导致应用程序启动。 这是一个统一的快速启动概念和标准的 Windows 任务栏。 实际上，不需要在 Windows 7 快速启动区域不再 ； 快速启动文件夹仍然处于 Windows 7 唯一原因是为向后兼容性。
您可能已经听说过有关跳转列表、 缩略图的工具栏、 任务栏进度栏和其他功能。 它们所有与该大型、 重新设计占用新任务栏的任务栏按钮。 图 2 中, 所示的 Windows 7 任务栏按钮被标识具有多个视觉效果。 它们能够立即识别正在运行的应用程序，是否在相同应用程序的应用程序当前处于活动状态并且的任务栏按钮当前有鼠标悬停在其上有多个窗口。

图 2 不同 Windows 7 任务栏按钮状态
所有 Windows 7 任务栏功能未锁定只有之后，都才为您的应用程序创建任务栏按钮。 这意义，因为如果不能更改任务栏覆盖图标渚嬪的方式  没有任务栏按钮还! 在练习，这意味着应用程序托管和本机应用程序使用新的 Windows 7 任务栏 API 将不得不等待新的窗口消息可识别 TaskbarButtonCreated 字符串。 要获得此消息的标识符，必须调用 RegisterWindowMessage Win32 API 和将 TaskbarButtonCreated 字符串作为参数传递。
以下代码显示了 Win32 应用程序接收窗口消息，指示创建的任务栏按钮的方式：
复制 
DWORD g_wmTBC = (DWORD)-1;

//In the window procedure:
switch (msg) {
case WM_CREATE:
    g_wmTBC = RegisterWindowMessage(L"TaskbarButtonCreated");
    //Proceed to create the window
case g_wmTBC:
    //The taskbar button has been created
}
托管应用程序，例如使用 Windows 窗体的应用程序将不得不重写来处理窗口消息窗体的窗口过程 (WndProc 方法）。 下面的代码演示此：
复制 
uint wmTBC = (uint)-1;

[DllImport("user32.dll")]
static extern uint RegisterWindowMessage(string message);

void Form_Load() {
    wmTBC = RegisterWindowMessage(L"TaskbarButtonCreated");
}
protected override void WndProc(...) {
    if (msg == wmTBC) {
        //The taskbar button has been created
    }
}
现在，您就可以知道如何确保您的应用程序不在任务栏按钮之前使用 Windows 7 任务栏。我们跳转到更有趣的功能之前，还有一个其他的条信息您需要完成填数游戏。如何任务栏按钮与相关联的窗口和应用程序？系统不会如何确定应用程序是否需要为其不同的窗口的两个任务栏按钮或多个应用程序是否需要共享一个任务栏按钮？
答案是应用程序 ID。这是新的属性的 Windows Shell，窗口、 进程和外壳程序的链接 （也称为快捷方式） 的属性。通过控制各个窗口和进程的应用程序 ID，应用程序开发人员可以确保任务栏按钮相关联的窗口完全他们认为适合。应用程序 ID 是一个字符串 — — 不是一个 GUID），可以采取最多 128 个字符的自由格式。要确保发生应用程序 ID 执行不会意外地冲突，建议是包含在应用程序 ID 公司名称、 产品的名称和应用程序的名称。默认为窗口应用程序 ID 取决于默认应用程序 ID 为属于窗口中的进程。这是，反过来，默认应用程序标识生成的可执行文件的过程运行。尝试启动应用程序希望记事本几次在操作中看到。没有创建，即使您在同一时间运行记事本的多个实例的只有一个任务栏按钮。
自定义这些默认 ID 涉及显式设置生成有趣的方案的应用程序 ID 为过程或一个单独的窗口。渚嬪的方式  如果将两个不同的进程的应用程序 ID 设置为相同的值，由这些过程的两个窗口会进行分组相同任务栏按钮。此外，如果将两个单独的窗口 （在同一进程中） 的应用程序 ID 设置为不同的值，两个任务栏按钮将会创建，另一个用于每个窗口。进程的应用程序 ID 和特定于窗口的应用程序 ID 的组合提供最大的灵活性。 图 3 ，在左的窗格显示从不同的进程窗口的所有分组在一个任务栏按钮。右窗格显示在同一进程中的多个窗口有多个的任务栏按钮。

图 3 Windows、 进程，和任务栏按钮
设置进程的应用程序 ID 涉及从 shell32.dll 对 SetCurrentProcessExplicitAppUserModelID Win32 函数调用。设置为窗口应用程序 ID 将需要调用 SHGetPropertyStoreForWindow 函数，然后操作返回的 IPropertyStore 对象。下面的示例演示如何执行此操作：
复制 
PROPVARIANT pv;
InitPropVariantFromString(L"MyAppID", &pv);

IPropertyStore* pps;
VERIFY(SHGetPropertyStoreForWindow(hwnd, IID_PPV_ARGS(&pps)));
VERIFY(pps->SetValue(PKEY_AppUserModel_ID, pv));
VERIFY(pps->Commit());

在 Windows API 代码为 Microsoft.NET 框架的包 提供可用于从托管代码访问新的 Windows 7 和 Windows Vista 功能的库。 . 在撰写本文时该代码包包括已知文件夹、 Windows Vista 任务对话框、 Windows 7 跳转列表、 图标贴，进度栏和其他功能的 Windows 7 澶栧 3 绋嬪簭库支持。 虽然这不是一个正式支持的 Microsoft 产品，该代码包提供一个很好的 jump-start。 托管应用程序开发人员针对 Windows 7 它 alleviates 需要实现复杂的 Windows 功能的托管的包装。
Windows API 的代码包提供用于操作给定的进程的应用程序 ID 的静态 Taskbar.AppId 属性。 遗憾的是，没有等效项设置的特定窗口应用程序 ID。 而是，这可以通过手动编写所需的包装或使用 Windows API 的代码包的前置任务完成。 （Windows 7 任务栏的互操作性示例库可以从下载 Windows 7 任务栏： 开发资源 . 使用库为您提供 Windows7Taskbar.SetWindowAppId 静态方法不会在难看的互操作性的工作）

覆盖图标和进度栏
大多数 Windows 7 任务栏功能路由是通过 ITaskbarList3 界面。 如果您使用托管的代码和托管的包装，您永远不可能要通过此接口提供。 但是，如果要使用本机代码，非常重要您知道如何获得一个手。 很简单，如果您只需确保这样做后，您收到该消息，指示已创建任务栏按钮。 换而言必须等待任务栏按钮会初始化，并获取指向该 ITaskbarList3 接口的指针，然后像下面这样：
复制 
ITaskbarList3* ptl;
VERIFY(CoCreateInstance(
    CLSID_TaskbarList, NULL, CLSCTX_ALL,
    IID_ITaskbarList3, (void**)&ptl));
设置覆盖图标，或提供 Windows 7 在任务栏中的一个进度栏是一项相当简单的任务。 幸运的是，托管开发人员，将托管等效项中没有 Windows API 的代码包。 若要设置覆盖图标，使用 ITaskbarList3::SetOverlayIcon 方法采用的图标句柄 （HICON） 和一个字符串，用作一个可访问的描述。 托管等同于 Taskbar.OverlayImage 属性同样易于使用。
下面的代码控制基于选定内容 （还显示了组合框中的应用程序的覆盖图标 图 4 ):
复制 
void comboBox1_ValueChanged(...) {
    if (comboBox1.SelectedItem == ...) {
        Taskbar.OverlayImage = new OverlayImage(icon1, "Icon 1");
    } else {
        Taskbar.OverlayImage = new OverlayImage(icon2, "Icon 2");
    }

下面的代码和图 6说明后台操作如何影响任务栏进度控件的状态 （假定在单独的的后台线程中运行 DoWork 函数）：

缩略图的工具栏
缩略图工具栏是一个很好 Windows 7 任务栏前景除了。缩略图和实时预览功能已被大大 enriched Windows 7 中 （遗憾的是，它不是在进行详细讨论这些本文的范围内）。缩略图的工具栏提供一个小的远程控制机会，从其缩略图窗口。渚嬪的方式  切换到下一个歌曲在 Windows 媒体播放器中不需要使用而言媒体播放机支持带区或切换到媒体播放器应用程序。您可以改用缩略图工具栏直接进行此项任务，无中断您的工作流的跳转到其他的应用程序，如 图 7 所示。

图 7 Windows 媒体播放器提供一个缩略图工具栏的 方便的导航。
若要用于一个缩略图工具栏，必须等待成为初始化，任务栏按钮然后再获取指向该 ITaskbarList3 接口的指针，然后如上文所述。下一步中，您初始化 THUMBBUTTON 结构描述个别的按钮显示在工具栏 （注意，仅限于最，七个按钮并有无法添加或删除按钮之后您初始化工具栏，只禁用或隐藏它们,） 中的一组。 最后，您调用 ITaskbarList3::ThumbBarAddButtons 和 ITaskbarList3::ThumbBarUpdateButtons 方法来控制缩略图工具栏。
单击一个缩略图按钮时, 应用程序收到 WM_COMMAND 消息的高的该 wParam 设置为 THBN_CLICKED 和低单词设置为按钮的 ID。此消息被调度到属于该缩略图窗口。 图 8 演示如何创建，并且需要处理的消息。

图 8 创建和消息处理

复制 
ITaskbarList3* ptl;//Created earlier

//In your window procedure:
switch (msg) {
case g_wmTBC://TaskbarButtonCreated
    THUMBBUTTON buttons[2];
    buttons[0].dwMask = THB_ICON|THB_TOOLTIP|THB_FLAGS;
    buttons[0].iId = 0;
    buttons[0].hIcon = GetIconForButton(0);
    wcscpy(buttons[0].szTip, L"Tooltip 1");
    buttons[0].dwFlags = THBF_ENABLED;
    buttons[1].dwMask = THB_ICON|THB_TOOLTIP|THB_FLAGS;
    buttons[1].iId = 1;
    buttons[1].hIcon = GetIconForButton(1);
    wcscpy(buttons[0].szTip, L"Tooltip 2");
    buttons[1].dwFlags = THBF_ENABLED;
    VERIFY(ptl->ThumbBarAddButtons(hWnd, 2, buttons));
    break;

case WM_COMMAND:
    if (HIWORD(wParam) == THBN_CLICKED) {
        if (LOWORD(wParam) == 0)
            MessageBox(L"Button 0 clicked", ...);
        if (LOWORD(wParam) == 1)
            MessageBox(L"Button 1 clicked", ...);
    }
    break;

跳转列表
，我们在此详述讨论 Windows 7 任务栏在最终功能就是跳转列表的认为最重要的功能应用程序需要利用为了真正 shine Windows 7。在 Windows 7 任务栏中每个任务栏按钮都与跳转列表关联。这是显示的项 （也称为目标） 的集合的 retractable 菜单和与给定的应用程序相关的任务。目标通常由外壳程序 （通过 IShellItem 接口） 的项目，您的应用程序使用的文件/文档。任务将由外壳程序的链接 （也称为快捷键，通过 IShellLink 界面），通常表示，并且可用于启动其他相关联的应用程序或甚至您自己的应用程序使用不同的命令行参数集。

托管等同得更容易使用。所有开头 Taskbar.JumpList 属性解除锁定的操作将跳转列表的目标和任务的方法： 并 AddToRecent 方法相当于 SHAddToRecentDocs KnownCategoryToDisplay 属性确定是否显示最新或频繁默认依据，UserTasks 属性返回用户的任务的集合，可修改然后 CustomCategories 属性返回自定义类别 （每个自定义类别有一个名称和跳转列表项的集合） 的集合。下面的示例演示用于向应用程序的跳转列表中添加项目和任务列表生成交易记录：
复制 
JumpListLink notepad = new JumpListLink {
    Title = "Launch Notepad", Path = @"C:\Windows\notepad.exe" };

JumpListItem doc1 = new JumpListItem(@"doc1.txt");
JumpListItem doc2 = new JumpListItem(@"doc2.txt");
CustomCategory category = new CustomCategory("Special Items");
category.JumpListItems.Add(doc1);
category.JumpListItems.Add(doc2);

Taskbar.JumpList.UserTasks.Add(notepad);
Taskbar.JumpList.CustomCategories.Add(category);
要注意的一个重要的是用户可以从任务栏删除项目，通过右键单击它们并选择"在此列表中删除。 应用程序必须知道此功能。濡傛它们灏濊瘯添加一个用户已删除的项目，它将导致整个列表生成事务失败。在 Win32 代码中，调用 ICustomDestinationList::BeginList 方法时，您收到应该小心额外，您不再次添加这些项目的已删除项的集合：

ICustomDestinationList* cdl;

IObjectArray* removedItems;
cdl->BeginList(&maxSlots, IID_IObjectArray, (void**)removedItems);
//…Iterate removedItems and make decisions
removedItems->Release();
或者，您可以主动调用 ICustomDestinationList::GetRemovedDestinations 方法启动列表生成交易记录之前检索已删除的项目。托管应用程序可以利用内置 Windows API 的代码包功能并且依赖于 JumpList 该类自动不会添加以前已由用户删除的项目。或者，托管应用程序可以注册到 Taskbar.JumpList.JumpListItemsRemoved 基础结构列表生成事务） 中检测到的项目已从跳转列表中删除时发生的事件。
